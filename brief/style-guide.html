05 · Radius scale

Tight. The angle does the work.

Curves are restrained. The chamfered corner is the system's hero geometry — not radius. Radius only on form inputs and the smallest auxiliary elements.

--ns-radius-none0 · buttons (chamfer)
--ns-radius-sm2px · small inline

06 · Shadow / depth

No shadows.

Depth comes from surface contrast (paper → paper-soft → anchor) and the chamfer cut, not from drop-shadows. The only exception is the focus ring (a 2px inset box-shadow on form fields).

07 · Motion language

Almost none.

Restrained editorial. Hover transitions only. No scroll-driven animation, no fade-in-on-scroll, no parallax. The site reads like a document.

23 · Image treatment

Real shop floor, real machines, real people.

No generic stock photography. No model-engineer staged shots. Real Kenworth machines, real Kenworth people, real machined parts (where NDA permits). Warm, natural light. No heavy grading, no teal-and-orange filter.

Open item — stock photography in use for now. The example crops below use stock CNC engineering imagery from assets/photography/. These will be replaced with real Kenworth shop-floor + machine + named-person photography before launch. Aspect-ratio rules and treatment guidance are locked; the photos themselves are placeholders. Flagged in client-strategy.md section 7.

+ Aspect ratios

+ Do

  • Real Kenworth machines, real parts, real people.
  • Warm, natural light. Slight contrast lift. Honest colour.
  • Captions on every image: machine name + what it does.
  • Use the chamfered corner on photo containers (matches button + card language).

+ Don't

  • No stock-photo "engineer holding tablet" shots.
  • No heavy duotone, no teal-and-orange grade, no faux-cinematic filter.
  • No staged smiling-team-around-a-machine photos.
  • Never crop a photograph in a way that loses the chamfer corner.

28 · Microcopy rules

Voice in the UI.

+ Button labels

  • Verb-led. "Send a drawing", "Get a quote", "Download capability sheet".
  • Sentence case. Never SHOUTY.
  • Never "Submit". Never "Click here". Never "Learn more".

+ Form placeholders

  • Real example. "natalie@company.co.uk", not "Email address".
  • Lowercase unless a proper noun.
  • Never repeat the label as the placeholder.

+ Error messages

  • What went wrong + what to do, in one sentence.
  • Specific. "That email's missing the @" beats "Invalid format".
  • Never blame the user.

+ Capitalisation + numbers

  • Sentence case for everything except headings (which can do title-tight where it reads better).
  • Numbers in tabular form on spec readouts. Units after a hair-space.
  • Tolerances: ≤ 10 µm (not 10µm or 10 micron).

30 · Visual signature — the chamfer

Where the angle lives.

The chamfered top-right corner is the project's distinctive design move. It appears on every button, every spec callout, every card and surface large enough to read it. It does not appear on body text containers, plain dividers, table cells, or anywhere it would shrink below 8px of cut.

31 · Accessibility

What's enforced.

  • Every palette pairing is contrast-tested. Body text on paper ≥ AAA. Body text on anchor ≥ AA.
  • Focus state is mandatory and visible on every interactive element: 2px primary-orange inset ring, never removed.
  • All images have alt text. Decorative SVGs use aria-hidden="true".
  • Keyboard navigation works through every component. Tab order follows visual order.
  • Respect prefers-reduced-motion — disable all transitions when set.
  • Form labels are always visible (not placeholder-only). Errors are announced via aria-live.
  • Buttons are button elements. Links are a. No "div pretending to be button".

32 · Performance

What's forbidden.

  • No autoplay hero video.
  • No carousel sliders.
  • No external JS animation libraries (GSAP, Lottie). CSS transitions only.
  • Hero image weight: ≤ 200KB. Card image weight: ≤ 80KB. WebP with JPEG fallback.
  • General Sans loaded with font-display: swap from Fontshare CDN.
  • LCP target: ≤ 1.8s. CLS target: ≤ 0.05. INP target: ≤ 200ms.

33 · The closed system

Do / Don't.

+ Do

  • Use orange only on conversion CTAs. CTA-only, never decorative.
  • Lead-left layouts. Hero copy ranged left, never centred.
  • Chamfered top-right corner on every button, spec callout, and primary card.
  • Tabular numerics on every spec readout.
  • Alternate paper / paper-soft / anchor — at least one anchor per page.
  • General Sans Display for headings, General Sans Variable for body.
  • Lucide stroke icons only, 1.75px weight, 24px default.
  • Real Kenworth machines and people in every photograph.

+ Don't

  • Don't use orange for body accents, borders, or icon fills.
  • Don't centre hero copy. Don't centre full sections.
  • Don't add a third typeface or a typewriter mono.
  • Don't add gradient text, gradient backgrounds, or neon palettes.
  • Don't use rounded buttons. Chamfer always.
  • Don't repeat 3-card icon grids section after section. Vary the rhythm.
  • Don't use stock photography. Don't use staged team photos.
  • Don't introduce shadows. Depth comes from surfaces.
